# 腾讯云智

作者：peakxy
修改时间：2026-05-11

## 一面（1h30min）

总部的面试官，感觉是直接跳过了真正意义上的一面。

### 实习

1. 简单介绍下学习和工作的经历？
2. 你做的这两个项目是自己感兴趣开发的还是团队一起参与的？
3. 本科期间大概学习的内容是什么？
4. 实习通过什么时候可以开始？
5. 能实习多久？
6. 聊一下你实习的业务。
7. 你们怎么盈利的？
8. 你们实习项目中一个请求处理的整体链路是什么？
9. 你们登录系统这一块怎么做的？
10. 讲一下实习中其他值得讲或者有亮点的地方。

### 项目

1. 你做的两个项目有想讲的吗？
2. 延迟队列和定时任务更新库存这块讲一下。
3. 发奖这一块怎么做的？

### 场景

1. 有一个函数，输入是一个字符串数组，每个数组元素是一个 `cd` 命令，这个 `cd` 命令就是 Linux 中切换目录的命令。你可以认为这些 `cd` 命令都是合法的，输出是 `pwd` 之后的结果，只关心最后的结果。让你实现这个函数的话，你有什么思路？
2. 浏览器有个前进后退的功能，让你去实现，你会怎么做？

### 八股

1. HTTP 和 HTTPS 的区别？
2. 非对称加密和对称加密是哪个算法，怎么配合的？
3. 对称加密用在握手阶段，那么怎么知道这个对称加密的密钥是什么呢？
4. 感觉这里面试官想问的是 TLS 四次握手的详细过程。

### 场景题

1. 我有一台 Linux 服务器，里面有两个文件 A、B，AB 文件都有 5000 万个字符串，每个字符串你可以认为是常见的 UUID。现在有个任务，要求去求它们的交集，然后把它们公共的部分输出到一个文件 C。还有一个限制，就是这台 Linux 的内存很少，连一个文件都放不下。
2. 12306 有一个典型的支付场景，就是抢到票之后，我不会马上支付，它会生成一个订单，这个订单 20 分钟后会超时关闭。这个你怎么设计？

### 开放

1. 你有什么想补充的点或者简历之外的点都可以补充？
2. 你为什么要做智能 OnCall 这样的系统呢？
3. 你平时用到这么多 token，是有学生优惠吗，还是学校本身提供的，纯属好奇。

### 反问

1. 业务：大数据组件、TCHouse、问数 Agent。
2. 学习建议：面试官说基础很扎实，没有什么好的建议，好奇你怎么学习的。
3. 一共几轮面试：正常情况下有 3 轮技术面。

## 二面（25min，总监面）

总部的面试官，大数据基础产品中心的总监。

1. 说一下你读研期间的研究方向。
   这块聊了 20 分钟。
2. 实习中印象最深的一点。
